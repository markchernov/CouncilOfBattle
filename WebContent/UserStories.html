User Stories

Student User Story #1

Student logs into the web app and is taken to a student desktop page. Here they want to be able to check their grades, attendance record, and enter a ticket for instructor assistance

Instructor User Story #1

An instructor logs into the web app and is taken to an instructor desktop page. From there they want to be able to enter/modify grades, update/modify the attendance record, view/modify tickets.

Instructor User Story #2

An instructor also wants to be able to create new projects, and assign students to them.

Admin User Story #1

The administrator logs into the web app and is taken to an admin desktop page. From here they want to be able to complete all CRUD interactions with all records.

Admin User Story #2

The administrator wants to be able to create cohorts, students and instructors accounts, give them access levels, update contact info, etc..

Admin User Story #3

The administrator wants to be able to create new subjects and assign competency

General User Story #1

CRUD can be simplified into three functions: a create function, a display function, and a modify function which includes editing and deleting. the .do processes should reflect this
